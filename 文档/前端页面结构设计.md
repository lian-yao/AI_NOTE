## 1. 当前前端定位

当前 `frontend/` 已替换为 React/Vite/Tauri 前端，不再使用 Vue、Element Plus、Pinia 或 `src/views` 路由页结构。

### 1.1 技术栈

| 类别 | 当前实现 |
|------|---------|
| 应用框架 | React 19 + TypeScript |
| 构建工具 | Vite 6 |
| 桌面端 | Tauri 2，桌面端使用 HashRouter |
| 样式 | Tailwind CSS 4 + 自定义暗色工作台视觉 |
| 基础组件 | Radix UI / shadcn 风格组件、Ant Design X、lucide-react |
| 状态管理 | Zustand |
| 请求层 | Axios 统一实例，启动健康检查使用原生 fetch |
| 内容渲染 | react-markdown、remark-gfm、Markmap |

### 1.2 路由与视图关系

React Router 只承担顶层入口；主功能在 `AppShell` 内通过内部 view 状态切换。

| 入口 | 组件 | 说明 |
|------|------|------|
| `/` | `AppShell` | 默认进入“新建”视图 |
| `/settings/*` | `AppShell initialView="settings"` | 直接进入设置视图 |
| `/onboarding` | `Onboarding` | Tauri 桌面端首启引导 |
| `*` | `NotFoundPage` | 未匹配路由 |

`AppShell` 内部视图：

| 视图 | 组件 | 说明 |
|------|------|------|
| 新建 | `GenerateView` | 链接/上传入口、模型选择、笔记风格和提示词 |
| 工作区 | `WorkspaceView` | 多标签笔记、总结、原文细读、字幕、思维导图、视频问答 |
| 知识库 | `LibraryView` | 历史任务、文件夹、搜索筛选、打开笔记 |
| 设置 | `SettingsView` | Provider、平台 Cookie、转写器、存储、运行状态 |

## 2. 页面结构

### 2.1 AppShell 布局

```text
+-------------------------------------------------------------+
|  左侧图标栏  |                 当前视图内容                  |
|              |                                               |
|  + 新建      |  GenerateView / WorkspaceView / LibraryView   |
|  工作区      |  / SettingsView                               |
|  知识库      |                                               |
|              |                                               |
|  设置        |                                               |
+-------------------------------------------------------------+
```

- 左侧栏固定宽度，使用图标按钮和 hover tooltip。
- 主区域通过 `currentView` 切换，非激活视图保持隐藏。
- 后端启动状态通过 `StartupBanner`、`BackendHealthIndicator`、`BackendInitDialog` 非阻塞提示。

### 2.2 新建视图

```text
+-------------------------------------------------------------+
| 看得更少，学得更多：加速音视频学习                           |
|                                                             |
| [链接] [上传]                                                |
| 输入 Bilibili 视频链接 / 上传本地文件                         |
| 模型选择、输出语言、音频语言、笔记风格                         |
|                                                             |
| 提示词内容 / 取个名字 / Prompt 快捷模板                         |
+-------------------------------------------------------------+
```

核心交互：

- 链接提交调用生成任务接口，并将任务加入 `taskStore`。
- 文件上传成功后以 `local` 平台继续提交生成任务。
- 后端未就绪时禁用提交按钮，但页面仍可浏览和配置。

### 2.3 工作区视图

```text
+-------------------------------------------------------------+
| 顶部标签栏：示例笔记 A | 示例笔记 B | ... | +                 |
+-----------------------------------------+-------------------+
| 总结 / 原文细读 / 字幕脚本 / 思维导图     | 视频预览 / 问答     |
| Markdown 内容、字幕分组、Markmap          | ChatPanel          |
+-----------------------------------------+-------------------+
```

核心交互：

- `openTasks` 控制顶部笔记标签，支持切换、关闭、相邻回退。
- 无真实任务时可使用开发 mock 数据预览标签效果。
- 成功任务展示总结、字幕和思维导图；非成功任务展示处理状态。
- 右侧面板承载视频信息和单视频问答。

### 2.4 知识库视图

```text
+-------------------------------------------------------------+
| 搜索 / 筛选 / 新建文件夹                                      |
|                                                             |
| 文件夹列表                 历史任务 / 笔记列表                 |
| 全部、收藏、最近等          状态、标题、时间、更多操作           |
+-------------------------------------------------------------+
```

核心交互：

- 从历史记录打开任务后进入工作区，并加入标签栏。
- 支持文件夹式组织和筛选入口。
- 当前阶段以前端状态和后端任务数据共同驱动。

### 2.5 设置视图

```text
+-------------------------------------------------------------+
| 模型接入 | 平台数据 | 本地转写 | 存储管理 | 运行状态           |
|                                                             |
| Provider / API Key / Models                                 |
| Bilibili Cookie / Proxy                                     |
| Whisper / Transcriber / 模型下载状态                         |
| Knowledge Base / Cache                                      |
| Backend / FFmpeg / CUDA                                     |
+-------------------------------------------------------------+
```

核心交互：

- Provider 支持增删改、启停、连接测试、拉取远程模型、启用模型。
- 平台数据管理 Bilibili Cookie 和代理。
- 本地转写管理转写器类型、Whisper 模型、下载状态和自定义模型映射。
- 运行状态展示后端、FFmpeg、CUDA、转写模型等诊断信息。

### 2.6 首启引导

`/onboarding` 仅在 Tauri 桌面端且未完成初始化时强制进入。Web 端不拦截用户。

## 4. 状态管理

当前使用 Zustand，而不是 Pinia。

| Store | 职责 |
|-------|------|
| `taskStore` | 任务列表、当前任务、任务内容更新、重试、删除 |
| `modelStore` | 已启用模型、远程模型、当前选中模型 |
| `providerStore` | Provider 列表、加载、增删改 |
| `chatStore` | 单任务问答历史和引用来源 |
| `configStore` | 存储路径和缓存目录配置 |

任务状态由 `useTaskPolling` 按间隔轮询，后续可与 WebSocket/SSE 事件合并。

---

## 5. 状态规范

| 状态 | 表现 | 当前实现位置 |
|------|------|-------------|
| loading | 按钮禁用、局部 loading、启动弹层 | `BackendInitDialog`、各视图本地状态 |
| empty | 图标/提示/行动按钮 | `WorkspaceView`、`LibraryView` |
| error | toast、诊断提示、重试入口 | Axios 拦截器、设置运行状态 |
| success | 正常渲染数据 | 各视图 |
| backend starting | 非阻塞提示 | `StartupBanner`、`BackendHealthIndicator` |

## 7. 设计规范

| 项 | 值 | 说明 |
|----|-----|------|
| 视觉基调 | 暗色工作台 | 当前主背景为 #0E0E0E / #111111 / #161616 |

## 8. 接口与运行配置

当前前端请求层由 `src/utils/request.ts` 统一创建 Axios 实例：

- `VITE_API_BASE_URL` 有值时直接作为后端基础地址。
- 未配置时默认使用 `/api`，由 `vite.config.ts` 代理到 `http://127.0.0.1:8000`。
- 前端开发端口由 `VITE_FRONTEND_PORT` 控制，默认 `5173`。

当前 `frontend/src/services/` 仍保留从 AI-video 前端迁移来的短路径契约；接口文档的目标契约为 `/api/v1/...`。联调阶段需要二选一：后端兼容短路径，或前端 services 统一迁移到 `/api/v1`。

| 功能 | 当前 services 路径 | 目标 API 契约 |
|------|-------------------|--------------|
| 新建任务 | `/generate_note`、`/upload` | `/api/v1/videos/process`、`/api/v1/uploads/videos` |
| 任务状态 | `/task_status/{task_id}` | `/api/v1/tasks/{task_id}` |
| 删除任务 | `/delete_task` | `/api/v1/videos/{video_id}` |
| 单视频问答 | `/chat/index`、`/chat/ask`、`/chat/status` | `/api/v1/qa/index`、`/api/v1/qa/ask`、`/api/v1/qa/index/status` |
| Provider / 模型 | `/get_all_providers`、`/add_provider`、`/model_list` 等 | `/api/v1/providers`、`/api/v1/models` |
| 转写器 / 平台配置 | `/transcriber_config`、`/whisper_models`、`/update_downloader_cookie`、`/proxy_config` | `/api/v1/transcribers/*`、`/api/v1/platforms/*`、`/api/v1/network/proxy` |
