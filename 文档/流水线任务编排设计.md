## 1. 设计目标

1. **阶段性**：每个阶段职责单一，可独立开发、测试、替换
2. **可恢复**：阶段失败后从断点重试，不丢失已完成中间产物
3. **可观测**：每个阶段的进度和状态可实时查询
4. **可控并发**：资源敏感阶段限制并发数

---

## 2. 流水线阶段

### 2.1 阶段总览

```text
[1. 链接解析]   -> meta.json
[2. 视频下载]   -> video.mp4
[3. 音频提取]   -> audio.wav（16kHz, mono）
[4. 语音转写]   -> transcription.json / .srt
[5. 笔记生成]   -> note.md
[6. 向量化存储] -> ChromaDB
```

### 2.2 阶段详情

| 阶段 | 模块 | 输入 | 产物 | 数据库变更 |
|------|------|------|------|-----------|
| parse | VideoProcessor | URL | meta.json | 创建 videos |
| download | VideoProcessor | URL | video.mp4 | 更新 status |
| extract | VideoProcessor | video.mp4 | audio.wav | 更新 audio_path |
| transcribe | Transcriber | audio.wav | transcription.json | 创建 tasks |
| generate | NoteGenerator | transcription | note.md | 创建 notes |
| store | VectorStore | note.md | ChromaDB 向量 | 创建 chunks |

### 2.3 状态机

```text
pending -> parsing -> parsed -> downloading -> downloaded
-> extracting -> extracted -> transcribing -> transcribed
-> generating -> generated -> storing -> completed
所有阶段都可进入 failed -> pending（重试）
```

## 3. 编排模型

### 3.1 PipelineOrchestrator

```python
class PipelineOrchestrator:
    def __init__(self):
        self.queue: asyncio.Queue[TaskContext] = asyncio.Queue()
        self.active_tasks: dict[str, TaskContext] = {}
        self.video_sem = asyncio.Semaphore(MAX_CONCURRENT)
        self.transcribe_sem = asyncio.Semaphore(1)

    async def submit(self, video_id: str) -> str:
        task_id = str(uuid.uuid4())
        ctx = TaskContext(task_id=task_id, video_id=video_id)
        await self.queue.put(ctx)
        self.active_tasks[task_id] = ctx
        return task_id

    async def _run_pipeline(self, ctx: TaskContext):
        stages = [
            ("parse", self._run_parse),
            ("download", self._run_download),
            ("extract", self._run_extract),
            ("transcribe", self._run_transcribe),
            ("generate", self._run_generate),
            ("store", self._run_store),
        ]
        for stage_name, stage_fn in stages:
            if ctx.should_skip(stage_name):
                continue
            success = await self._run_stage(ctx, stage_name, stage_fn)
            if not success:
                break
```

## 4. 并发控制

| 资源 | 瓶颈 | 策略 |
|------|------|------|
| GPU 显存 | Whisper 模型 ~2 GB | semaphore = 1（串行转写）|
| 磁盘 I/O | 下载 + 转写同时读写 | 不同时执行 |
| B 站限流 | 频繁下载被 ban | 请求间隔 >= 10s |

## 5. 错误处理

### 5.1 重试策略

| 阶段 | 可重试 | 策略 | 上限 |
|------|--------|------|------|
| parse | 否 | - | 0 |
| download | 是 | 指数退避 5s -> 25s -> 125s | 3 |
| extract | 是 | 固定 5s | 2 |
| transcribe | 是 | 切换到 CPU 重试 | 2 |
| generate | 是 | 固定 10s | 3 |
| store | 是 | 固定 5s | 3 |

### 5.2 降级方案

| 阶段 | 降级 | 条件 |
|------|------|------|
| download | 降级到 480p | 720p+ 下载失败 |
| transcribe | 切换到必剪 API | 本地 Whisper 连败 2 次 |
| generate | 切换到 DeepSeek | 通义 API 连败 |
| store | 跳过向量化 | ChromaDB 写入失败 |

### 5.3 重启恢复

```python
class PipelineRecovery:
    @staticmethod
    async def recover(db_session):
        # 1. running -> pending（进程被 kill）
        for t in db_session.query(Task).filter(Task.status == "running").all():
            t.status = "pending"
            t.error_message = "进程重启，任务已重置"
        # 2. pending 重新入队
        for t in db_session.query(Task).filter(Task.status == "pending").all():
            if ArtifactChecker.check(t.type, get_video_dir(t.video_id)):
                t.status = "completed"
            else:
                await orchestrator.resubmit(t.video_id, resume_from=t.type)
        db_session.commit()
```

## 6. 中间产物

### 6.1 目录结构

```text
data/videos/{video_id}/
  meta.json               # 解析结果
  video.mp4               # 视频文件
  audio.wav               # 音频文件
  transcription.json      # 转写结果（JSON 带时间戳）
  transcription.srt       # 转写结果（SRT 字幕）

```

### 6.2 文件清理

| 时机 | 清理内容 | 条件 |
|------|---------|------|
| 笔记生成后 | transcription.* | 可选 |
| 音频提取后 | video.mp4 | video_retention != "always" |
| 处理完成 | audio.wav | video_retention == "processed" |
| 删除视频 | 整个目录 | 用户操作 |

## 7. 进度通知

| 阶段 | 方式 | 频率 |
|------|------|------|
| parse | 一次性 | 0/100 |
| download | yt-dlp hook | 每 ~5% |

| generate | 开始/结束 | 50%/100% |
| store | 切片计数 | 每 10 个 |

### 7.1 事件总线

```python
@dataclass
class PipelineEvent:
    event: str       # progress / completed / error
    task_id: str
    video_id: str
    stage: str
    status: str      # running / completed / failed
    progress: int    # 0-100
    message: str


class EventBus:
    def __init__(self):
        self._subscribers: dict[str, list[Callable]] = defaultdict(list)

    async def emit(self, event: PipelineEvent):
        for cb in self._subscribers.get(event.task_id, []):
            await cb(event)


event_bus = EventBus()
```

class VideoParser(ABC):
