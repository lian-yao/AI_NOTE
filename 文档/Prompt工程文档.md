## 1. 概述

### 1.1 Prompt 在本项目中的作用

| 场景 | LLM 任务 | 质量影响 |
|------|---------|---------|
| 笔记生成 | 转写文本 -> 结构化 Markdown | 笔记的可读性、准确性 |
| 智能问答 | 检索上下文 + 问题 -> 回答 | 回答准确性和来源可信度 |

### 1.2 当前使用的模型

| 模型 | 上下文窗口 | 特点 |
|------|-----------|------|
| qwen-plus（通义千问） | 131,072 tokens | 中文理解强，指令遵循稳定 |
| deepseek-chat（DeepSeek） | 64,000 tokens | 推理能力强，价格低 |

> DeepSeek 的 System Prompt 遵循度不如通义，关键约束需在 User Prompt 中重复。

## 2. 笔记生成 Prompt

### 2.1 System Prompt

```text
你是一个专业的视频笔记整理助手。你的任务是将视频的语音转写文本转化为结构清晰、
内容完整的 Markdown 笔记。

## 核心要求
1. 忠实原文：只基于转写文本中的内容
2. 结构化：识别视频自然语义分段，按章节整理
3. 时间标注：每个章节标注起止时间
4. 完整性：覆盖所有重要知识点
5. 客观性：保持中立语气

## 输出格式
# {视频标题}

## 摘要
用 2-4 句话概括核心内容。

## 关键词
- 关键词1
- 关键词2

## 内容整理
### 章节一：{标题}（MM:SS - MM:SS）
{内容整理}

## 核心观点总结
1. ...
2. ...

## 金句摘录
> 精彩语句
> -- 出处（MM:SS）
```

### 2.2 User Prompt

```text
请根据以下视频信息，生成结构化笔记。

## 视频信息
- 标题：{video_title}
- UP 主：{uploader}
- 时长：{duration} 秒

## 转写文本
{transcript_text_with_timestamps}

请按上述要求生成 Markdown 笔记。
```

### 2.3 Token 预算

| 视频时长 | 转写文本 | 约 token |
|---------|---------|---------|
| 10 分钟 | ~2,000 字 | ~3,000 |
| 30 分钟 | ~6,000 字 | ~8,000 |
| 60 分钟 | ~12,000 字 | ~16,000 |
| 2 小时 | ~24,000 字 | ~32,000 |

## 3. 智能问答 Prompt

### 3.1 System Prompt

```text
你是一个基于知识库的智能问答助手。

## 核心原则
1. 只使用提供的参考资料来回答问题
2. 参考资料中没有相关信息时，明确告知未找到
3. 在回答中标注引用来源（视频标题 + 章节）
4. 用中文回答，语言简洁易懂

## 引用格式
> 引用内容
> -- 来源：《视频标题》- 章节名称
```

### 3.2 User Prompt

```text
## 参考资料
{retrieved_chunks_formatted}

---

## 问题
{user_query}

请基于参考资料回答上述问题。
```

## 4. 模型适配

### 4.1 通义千问
- 温度：0.1（笔记生成）/ 0.3（问答）
- Markdown 格式输出稳定
- 长文本（> 50K tokens）可能出现重复，建议分段

### 4.2 DeepSeek
- 温度：0.1（笔记生成）/ 0.3（问答）
- System Prompt 遵循度一般，关键指令需在 User Prompt 重复
- 长文本偶尔出现幻觉，需强调只基于参考资料

## 5. 已知问题

| 问题 | 表现 | 缓解 |
|------|------|------|
| 遗漏章节 | 笔记章节偏少 | prompt 强调覆盖所有知识点 |
| 格式偏移 | Markdown 结构不标准 | User Prompt 末尾重复格式要求 |
| 幻觉 | 回答增加原文没有的信息 | 降低 temperature |
| 长视频中间丢失 | 2h+ 视频中间部分质量下降 | 分段处理（待实现） |
