## 2. 后端目录（app/）

### 2.1 目录总览

```
app/
├── api/v1/       # FastAPI 路由层，5 组 RESTful 接口
├── core/         # 基础设施：配置、错误处理、日志
├── llm/          # LLM 统一客户端（通义千问 + DeepSeek）
├── models/       # SQLAlchemy ORM 数据模型（5 个表）
├── note/         # 笔记生成器（LLM prompt + Markdown 解析）
├── pipeline/     # 流水线编排 + 事件总线
├── processor/    # 视频处理：解析/下载/音频提取
├── qa/           # 智能问答引擎
├── retriever/    # 混合检索器（向量 + 关键词 + RRF）
├── schemas/      # Pydantic 请求/响应模型
├── store/        # 向量化存储（切片 + embedding + ChromaDB）
└── transcriber/  # 语音转写（Whisper + 必剪 API）
```

### 2.2 各包说明

| 目录 | 职责 | 对应文档中的模块 | 输入 | 输出 |
|------|------|----------------|------|------|
| api/v1/ | HTTP 接口路由，参数校验，响应组装 | API 接口文档 | HTTP 请求 | HTTP 响应 |
| core/ | 配置加载、全局异常处理器、日志初始化 | 技术方案 6/7 节 | 环境变量/config.yaml | 配置对象 |
| llm/ | 统一封装 LLM chat 和 stream_chat 调用 | Prompt 工程文档 4 节 | messages 列表 | 文本/流式生成 |
| models/ | SQLAlchemy ORM 声明式映射 | 数据库设计文档 7 节 | - | ORM 模型类 |
| note/ | 构建 prompt 调 LLM 生成笔记，解析输出 | 技术方案 3.3 节 | 转写文本 + 元数据 | StageResult |
| pipeline/ | 编排各模块的执行顺序、并发控制、事件推送 | 流水线任务编排设计 | TaskContext | PipelineEvent |
| processor/ | yt-dlp 下载、ffmpeg 提取音频、URL 解析 | 技术方案 3.1 节 | 视频 URL | 视频/音频文件 |
| qa/ | 检索增强问答上下文构建 + LLM 流式回答 | 技术方案 3.5 节 | 问题 + video_id | 流式回答 + 引用 |
| retriever/ | 向量相似度 + 关键词 + RRF 排序融合 | 技术方案 3.5 节 | 问题文本 | 排序后的切片列表 |
| schemas/ | Pydantic 模型定义，API 请求/响应序列化 | API 接口文档 8 节 | - | Pydantic BaseModel |
| store/ | 按标题语义切片、text-embedding-v3 向量化、ChromaDB 读写 | 技术方案 3.4 节 | note.md | ChromaDB 向量 |
| transcriber/ | 策略模式封装 Faster-Whisper 和必剪 API | 技术方案 3.2 节 | audio.wav | transcription.json |

### 2.3 模块依赖关系

```
api/v1/  (路由层，无业务逻辑)
  ├── 依赖 pipeline/ 提交视频处理
  ├── 依赖 store/ 查询向量
  └── 依赖 qa/ 处理问答

pipeline/  (编排层，调度各模块)
  ├── processor/  下载视频、提取音频
  ├── transcriber/ 转写音频
  ├── note/       生成笔记
  └── store/      向量化存储

qa/  (问答层)
  ├── retriever/  检索相关片段
  ├── llm/        调用大模型生成回答
  └── store/      查询向量库
```

## 3. 前端目录（frontend/）

### 3.1 目录总览

```
frontend/
├── public/               # 静态资源与预览图
├── src-tauri/            # Tauri 2 桌面端配置、Rust 入口、权限与图标
├── src/
│   ├── components/       # React 通用组件、健康状态、图标、shadcn 风格 UI
│   ├── pages/            # 页面与 AppShell 主工作台
│   ├── services/         # Axios API 封装
│   ├── store/            # Zustand 状态管理
│   ├── hooks/            # 后端就绪检查、任务轮询等 Hook
│   ├── constant/         # 笔记格式、平台选项等常量
│   ├── lib/              # 工具函数与 Markmap 适配
│   ├── types/            # TypeScript 类型定义
│   ├── utils/            # 请求实例、通用工具
│   ├── App.tsx           # 路由、后端状态、桌面端 onboarding 守卫
│   └── main.tsx          # React 应用入口
├── index.html
├── vite.config.ts
├── components.json
├── postcss.config.cjs
├── tailwind.config.cjs
├── tsconfig*.json
├── package-lock.json
└── package.json
```

### 3.2 各目录说明

| 目录 | 内容 | 说明 |
|------|------|------|
| pages/AppShell/ | GenerateView、WorkspaceView、LibraryView、SettingsView、ShellSidebar | 主应用四视图与侧边栏 |
| pages/Onboarding/ | 桌面端首启配置页 | Tauri 环境下未完成初始化时进入 |
| components/ui/ | button、dialog、select、tabs、switch、tooltip 等 | Radix/shadcn 风格基础组件 |
| components/BackendHealth/ | 后端健康状态、日志面板、事件监听 | 启动期与运行期状态反馈 |
| services/ | note、chat、model、system、transcriber、proxy、upload 等 API 封装 | 基于 `utils/request.ts` 的 Axios 实例 |
| store/ | taskStore、modelStore、providerStore、chatStore、configStore | Zustand Store |
| hooks/ | useCheckBackend、useTaskPolling | 后端就绪检查与任务轮询 |
| src-tauri/ | Cargo、tauri.conf、capabilities、Rust 入口 | Tauri 2 桌面端外壳 |

### 3.3 页面路由

| 页面 | 路由 | 说明 |
|------|------|------|
| 新建 | / | 链接输入、上传、本次生成参数、模型选择 |
| 工作区 | / | AppShell 内部视图：已打开笔记标签、总结、字幕、思维导图、视频问答 |
| 知识库 | / | AppShell 内部视图：文件夹、历史任务、搜索筛选 |
| 设置 | /settings/* | Provider、平台 Cookie、转写器、存储、运行状态 |
| 首启引导 | /onboarding | Tauri 桌面端 onboarding |

## 4. 数据目录（data/）

```
data/
├── chromadb/    # ChromaDB 向量数据库持久化文件
├── logs/        # 运行日志（按日期轮转）
└── videos/      # 视频处理中间产物
    └── {video_id}/
        ├── meta.json           # 解析阶段 -> 元数据
        ├── video.mp4           # 下载阶段 -> 视频文件
        ├── audio.wav           # 提取阶段 -> 音频（16kHz mono）
        ├── transcription.json  # 转写阶段 -> 带时间戳文本
        ├── transcription.srt   # 转写阶段 -> 字幕文件
        └── note.md             # 生成阶段 -> 结构化笔记
```
